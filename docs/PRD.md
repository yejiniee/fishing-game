# 🐟 방어잡기 — 게임 기획서

## 1. 프로젝트 개요

| 항목       | 내용                                                                  |
| ---------- | --------------------------------------------------------------------- |
| 프로젝트명 | 방어잡기 (Catch the Yellowtail)                                       |
| 장르       | 캐주얼 반사신경 / 타이밍 웹게임                                       |
| 플랫폼     | 웹 브라우저 (데스크톱 & 모바일 반응형)                                |
| 기술 스택  | React + TypeScript + Tailwind CSS + GSAP + Vite                       |
| 타깃       | 짧은 시간에 순발력과 판단력을 시험하는 캐주얼 게임을 원하는 전 연령층 |

### 컨셉

방어는 물 밖에서 팔딱팔딱 뛰며 가만있지 않는다. 화면 곳곳에 등장하는 방어를, 그물 링이 다 줄어들기 전에 재빨리 탭해서 잡아야 한다. 그런데 방어와 함께 **복어·오징어**도 섞여 나온다 — 이 둘은 잡으면 안 되는 미끼로, 잘못 건드리면 목숨이 깎인다. 잡을수록 방어가 더 빨리, 더 많이 등장해 난이도가 점점 올라가고, 최종적으로 잡은 마릿수에 따라 우스꽝스러운 칭호가 붙는다.

---

## 2. 핵심 게임플레이 루프

```
게임 시작 → 방어/복어/오징어가 무작위 위치에 등장(그라운드 상태)
   → 그물 링이 줄어드는 동안 탭 여부 판정
      ├─ 방어를 탭 → 점수·마릿수·콤보 증가, 새 자리로 재등장
      ├─ 복어·오징어를 탭 → 목숨 -1 (페널티), 새 자리로 재등장
      ├─ 방어를 시간 안에 못 잡음 → 목숨 -1, 다른 자리로 도주
      └─ 복어·오징어를 안 건드림 → 페널티 없이 조용히 다른 자리로 이동
   → 목숨 0 → 게임 오버, 칭호·점수·최고 기록 표시
      ├─ 닉네임 입력(최초 1회) → 전역 랭킹에 내 최고 기록 등록
      ├─ 랭킹 보드에서 TOP N + 내 순위 확인
      └─ 결과 카드 이미지로 저장/공유
        ↑                                                      |
        └────────────────── 다시하기 ←─────────────────────────┘
```

## 3. 핵심 기능 명세

### 3.1 등장 생물 시스템

| 종류                 | 등장 비율 | 잡았을 때                                | 색상·표정                                                  |
| -------------------- | --------- | ---------------------------------------- | ---------------------------------------------------------- |
| 🐟 방어 (yellowtail) | 68%       | 점수 획득 + 마릿수/콤보 증가 (진짜 목표) | 항상 파란색으로 고정, **크기·표정만** 무작위 (헷갈림 방지) |
| 🐡 복어 (pufferfish) | 18%       | **목숨 -1** (미끼, 잡으면 안 됨)         | 둥글고 가시 돋친 실루엣, 색상·표정 고정                    |
| 🦑 오징어 (squid)    | 14%       | **목숨 -1** (미끼, 잡으면 안 됨)         | 몸통 + 촉수 실루엣, 색상·표정 고정                         |

- 방어는 색을 랜덤화하면 미끼와 헷갈릴 수 있어 단일 색으로 고정하고, 종(species)의 실루엣 차이만으로 구분하게 했다.
- 표정: 방어는 평범/행복/놀람/졸림 4종 중 무작위. 복어·오징어는 각자 고정된 표정(놀람/능청)으로 항상 같은 인상을 준다.

### 3.2 크기 시스템

| 크기       | 화면 배율 | 방어 점수 | 그라운드 지속시간 배율 |
| ---------- | --------- | --------- | ---------------------- |
| 대(large)  | 1.35배    | 1점       | 1.2배 (느긋)           |
| 중(medium) | 1배       | 2점       | 1배                    |
| 소(small)  | 0.7배     | 3점       | 0.82배 (다급)          |

- 등장할 때마다 대/중/소 중 하나가 30% / 40% / 30% 확률로 뽑힌다.
- 작을수록 화면 판정 영역이 작아 잡기 어려운 대신 점수가 높고, 그물 링도 더 빨리 줄어든다.
- 복어·오징어도 같은 크기 배율을 공유해 시각적 다양성을 준다 (단, 점수에는 영향 없음).

### 3.3 잡기(캐치) 메커니즘 — 그물 링

- 생물이 새 자리에 착지하면 몸을 파닥거리며 **그물 링**이 초록 → 노랑 → 빨강으로 줄어드는 카운트다운이 시작된다.
- 링이 다 줄어들기 전에 탭하면 판정, 다 줄면 자동으로 다른 자리로 튀어 나간다(도주).
- 판정 결과:
  - 방어를 탭 → 성공(+점수), 방어를 놓침(시간초과) → 실패(목숨 -1)
  - 복어·오징어를 탭 → 실패(목숨 -1), 안 건드림(시간초과) → 안전 (올바른 선택)

### 3.4 점수·목숨·콤보

- 시작 목숨: 5개. 0이 되면 게임 오버.
- 방어를 잡으면: 점수 += 크기별 점수, 잡은 마릿수(catchCount) +1, 콤보 +1, 콤보 3 이상이면 HUD에 "🔥 N 콤보" 표시.
- 방어를 놓치거나 미끼를 잘못 잡으면: 목숨 -1, 콤보 0으로 초기화.
- 최고 점수는 `localStorage`에 저장되어 새로고침해도 유지된다.

### 3.5 난이도 스테이지 — 점진적 난이도 상승

잡은 마릿수(catchCount, 미끼 제외) 기준으로 5단계 난이도가 있다. 단계가 올라가면 동시 등장 마릿수가 늘고, 그라운드 지속시간과 점프 시간이 짧아진다. 같은 단계 안에서도 잡을수록 그라운드 지속시간이 계속 짧아진다.

| 단계 진입 마릿수 | 동시 등장 마릿수 | 그라운드 기본/최저 시간 | 점프 시간 |
| ---------------- | ---------------- | ----------------------- | --------- |
| 0마리            | 3마리            | 1650ms → 520ms          | 420ms     |
| 8마리            | 4마리            | 1500ms → 460ms          | 380ms     |
| 20마리           | 4마리            | 1300ms → 400ms          | 340ms     |
| 30마리           | 5마리            | 1150ms → 360ms          | 300ms     |
| 70마리           | 5마리            | 1000ms → 320ms          | 260ms     |

### 3.6 칭호 시스템

게임 오버 화면에 잡은 마릿수 기준 칭호가 뜬다. 다음 칭호까지 남은 마릿수도 함께 안내한다.

| 잡은 마릿수 | 칭호                      |
| ----------- | ------------------------- |
| 0 ~ 30마리  | 방어잡다 번아웃 온 사장님 |
| 31 ~ 40마리 | 동네 횟집 사장님          |
| 41 ~ 50마리 | 오마카세 사장님           |
| 51마리 이상 | 방어잡기의 신             |

### 3.7 피드백 연출

| 상황                 | 토스트                                         | 파티클    |
| -------------------- | ---------------------------------------------- | --------- |
| 방어 포획 성공       | 초록 토스트, "+N 찰싹! 잡았다!" 등 랜덤 메시지 | 💦 물보라 |
| 방어 놓침 (시간초과) | 주황 토스트, "미끄덩... 놓쳤다!" 등            | 💨        |
| 미끼 잘못 포획       | 빨강 토스트, "⚠️ 앗, 그건 복어잖아!" 등        | 💥        |

### 3.8 랭킹 시스템 — 전역 랭킹

게임 오버 시 잡은 마릿수(catchCount)를 기준으로 전역 랭킹에 기록을 남기고, 다른 플레이어와 순위를 겨룬다. 별도 로그인 없이 참여할 수 있다.

- **식별:** 최초 게임 오버 시 닉네임(2~8자)을 한 번 입력받고, 기기별로 발급한 UUID(`localStorage` 저장)에 연결한다. 이후에는 같은 이름으로 자동 등록되며, 닉네임은 언제든 변경할 수 있다.
- **기록 갱신:** 기기 ID(`device_id`) 기준 **upsert**로 본인 최고 기록만 유지한다. 새 점수가 기존 최고 기록보다 높을 때만 갱신되고, 낮으면 무시된다.
- **표시:** 랭킹 보드에 상위 N명(예: TOP 20)을 순위·닉네임·마릿수·점수로 보여주고, 내 순위는 하이라이트한다. 목록에 들지 못하면 하단에 "내 순위 NN위"를 별도로 안내한다.
- **정렬 기준:** 잡은 마릿수 내림차순, 동률이면 점수 → 등록 시각 순.
- **예외 처리:** 네트워크 실패 시 랭킹 등록/조회가 안 돼도 게임 진행과 로컬 최고 점수에는 영향이 없도록 하고, 재시도 안내를 띄운다.

### 3.9 결과 공유 — 이미지 카드

게임 오버 결과를 이미지 카드로 만들어 저장·공유할 수 있다.

- **카드 내용:** 최종 점수, 잡은 마릿수, 칭호, 게임 로고/캐릭터를 담은 정사각(또는 세로형) 이미지.
- **동작:** `공유` 버튼을 누르면 카드 이미지를 생성해 **Web Share API**로 시스템 공유 시트를 띄운다(모바일 인스타·카카오톡 등). Web Share API를 지원하지 않는 환경에서는 **이미지 다운로드**로 대체한다.
- **생성 방식:** 결과 카드 DOM을 캔버스로 렌더링해 PNG로 변환한다(예: `html-to-image`/`canvas` 계열). 외부 서버 없이 클라이언트에서 처리한다.

## 5. 기술 설계 개요

### 5.1 상태 관리 (React State)

```ts
interface FishEntity {
  id: number;
  phase: "grounded" | "jumping" | "caught";
  position: { x: number; y: number }; // % 좌표
  kind: "yellowtail" | "pufferfish" | "squid";
  size: "small" | "medium" | "large";
  expression: "normal" | "happy" | "surprised" | "sleepy";
  round: number; // 개체별 애니메이션 재시작 트리거
}

interface GameState {
  phase: "ready" | "playing" | "gameover";
  fish: FishEntity[]; // 동시에 여러 마리 독립 관리
  score: number;
  catchCount: number; // 칭호/난이도 스테이지 기준
  best: number; // localStorage 저장
  lives: number;
  combo: number;
  feedback: Feedback | null;
}

interface RankingEntry {
  id: string;
  deviceId: string; // 기기별 UUID (localStorage)
  nickname: string; // 2~8자
  score: number;
  catchCount: number;
  createdAt: string;
}
```

- 개체(FishEntity)마다 자기 자신의 그라운드→점프→착지 타이머를 독립적으로 갖는다 (`setTimeout` 기반, `useEffect` 없이 액션 함수 안에서 다음 타이머를 직접 예약하는 방식).
- 난이도 스테이지가 올라 동시 등장 마릿수가 늘면, 부족한 만큼 새 개체를 즉시 스폰한다.

### 5.4 전역 랭킹 백엔드 (Supabase)

랭킹은 **Supabase**(Postgres + JS 클라이언트)에 저장한다. 별도 서버 코드 없이 SPA에서 직접 read/upsert 하고, 보안은 RLS(Row Level Security)로 제어한다.

```sql
-- scores 테이블
create table scores (
  id          uuid primary key default gen_random_uuid(),
  device_id   uuid not null unique,      -- 기기별 식별, upsert 키
  nickname    text not null,             -- 2~8자
  score       integer not null,
  catch_count integer not null,
  created_at  timestamptz not null default now()
);
create index scores_rank_idx on scores (catch_count desc, score desc, created_at asc);
```

- **기기 ID:** `lib/deviceId.ts`가 `localStorage`에 UUID를 생성·보관한다. 랭킹 등록·조회의 식별자로 사용한다.
- **API 계층:** `lib/ranking.ts`에 `submitScore()`(device_id 기준 upsert, 기존 최고 기록보다 높을 때만 갱신)와 `fetchTopScores(limit)`(정렬 조회)를 둔다.
- **보안(RLS):** `select`와 본인 `device_id` 대상 `insert/update`만 허용하고, 타인 기록 수정·삭제는 차단한다. 닉네임 길이·문자를 클라이언트와 DB 제약 양쪽에서 검증한다. Supabase anon key는 클라이언트에 노출되어도 되며(RLS가 실제 방어선), URL/키는 환경변수(`.env`, `VITE_` 접두사)로 관리한다.
- **오프라인/실패:** 네트워크 오류 시 게임 진행과 로컬 최고 점수는 그대로 두고, 랭킹 등록/조회만 실패 안내 + 재시도로 처리한다.

### 5.5 결과 공유 (클라이언트 이미지 생성)

- `<ShareCard />`의 결과 카드 DOM을 캔버스로 렌더링해 PNG로 변환한다(`html-to-image` 등 클라이언트 라이브러리, 외부 서버 없음).
- `공유` 버튼 → `navigator.share`로 이미지 파일 공유. 미지원 환경은 PNG 다운로드로 대체(`navigator.canShare` 분기).

### 5.3 애니메이션 (GSAP)

| 대상           | 애니메이션                                                                                         |
| -------------- | -------------------------------------------------------------------------------------------------- |
| 그라운드(대기) | 몸통 squash/stretch + 꼬리(또는 촉수/지느러미) 파닥임 반복. 크기가 작을수록 더 빠르게 파닥인다     |
| 점프(이동)     | 도약 스쿼시 → 체공 아크 → 착지 바운스. 위치 이동은 CSS `transition`으로, 세부 모션은 GSAP으로 처리 |
| 포획/페널티    | 순간적으로 움츠러들며 스케일 축소 + 페이드아웃                                                     |
| 그물 링        | CSS `stroke-dashoffset` 애니메이션으로 색이 초록→빨강으로 바뀌며 줄어듦                            |
| 파티클         | 랜덤 방향으로 튀어나가며 페이드아웃 후 DOM 제거 (GSAP timeline)                                    |
| 토스트         | fade-in + scale → 유지 → fade-out (CSS keyframe)                                                   |

## 6. 개발 범위 (MVP)

**포함 (본 기획서 범위)**

1. 방어/복어/오징어 렌더링 (종별 SVG, 크기·표정 변주)
2. 그물 링 타이밍 기반 포획 메커니즘 (성공/실패/페널티 판정)
3. 점수 · 목숨 · 콤보 · 잡은 마릿수 관리
4. 난이도 스테이지 (동시 등장 마릿수 증가 + 그라운드/점프 시간 단축)
5. 칭호 시스템 (4단계, 게임 오버 화면 표시)
6. 최고 점수 저장 (localStorage)
7. 시작/게임오버 화면, 피드백 토스트·파티클 연출
8. 전역 랭킹 (Supabase, 닉네임 + 기기 ID 식별, 본인 최고 기록 upsert, TOP N 표시)
9. 결과 공유 (결과 카드 이미지 생성 + Web Share API / 다운로드)

**제외 (향후 확장 아이디어)**

- 사운드 이펙트
- 계정 로그인 / 소셜 인증
- 추가 미끼·보너스 생물 종류 (예: 황금 방어, 폭탄 등)
- 일일 도전 과제, 스킨/커스터마이징
